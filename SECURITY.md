# Security policy

## Supported download

Only the beta version and SHA-256 checksum published on the Sift download page are supported. A file with a different checksum should be treated as untrusted.

## Report a vulnerability privately

Use [GitHub private vulnerability reporting](https://github.com/n3try/sift-ai-video-filter-site/security/advisories/new) for security flaws in the website or downloadable extension. Do not publish exploit details, tokens, installation identifiers, private browsing data, or account information in a public issue.

Include the affected version, browser and operating system, reproduction steps, security impact, and the least sensitive proof needed to explain the problem. Do not test against accounts, videos, systems, or third-party services you do not own or have permission to assess.

This project does not currently offer a bug bounty. A private report is still useful, but no payment or response deadline is promised.

Accessibility barriers that do not expose data or create a security risk can be reported through the public issue route described on `accessibility.html`.
