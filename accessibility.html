Accessibility statement

Sift should work for more than one way of browsing.

The Sift website is being developed toward WCAG 2.2 Level AA. It has not received an independent accessibility audit, and this statement is not a claim of legal certification.

1. Commitment and standard

Sift aims to provide equal access to its download, installation, privacy, terms, and support information. WCAG 2.2 Level AA is the project target because it provides a concrete technical baseline for perceivable, operable, understandable, and robust web content.

Automated results alone cannot establish accessibility conformance. The status above will remain "Verification in progress" until the manual and assistive-technology checks listed below are completed and any resulting barriers are fixed.

2. Current accessibility features

  • Semantic headings, landmarks, lists, tables, and descriptive page titles.
  • Skip links that move keyboard focus to the main content.
  • Visible keyboard focus and no positive tab order overrides.
  • Text and controls designed to reflow on narrow screens.
  • Reduced-motion and forced-colors styles.
  • Persistent status text after the checksum copy action.
  • Keyboard scrolling instructions and arrow-key support for the wide permissions table.

3. Testing completed

The project source checker currently passes all published pages. It checks page structure, accessible names, label matching, table headers, fragment targets, focus treatment, reduced motion, responsive rules, selected color-contrast pairs, persistent status messages, and keyboard support for the permissions table.

The current automated gate does not render these pages or exercise their JavaScript in a browser. Browser results will not be treated as release evidence until the browser, version, operating system, date, and findings are recorded.

What this does not prove

A passing automated source check does not prove that every user can complete every task with assistive technology.

4. Verification still required

  • Complete every page and action using only a keyboard and confirm a logical focus order.
  • Verify text resizing at 200 percent and page reflow at 400 percent without lost content or controls.
  • Test Windows High Contrast and browser forced-colors modes.
  • Test current Chrome with NVDA on Windows.
  • Test current Safari with VoiceOver on macOS.

Until these checks pass, Sift does not describe this website as audited, certified, or fully conformant.

5. Report an accessibility barrier

If part of this website prevents you from downloading Sift or reading its information, open an issue in the Sift support tracker. Start the issue title with "Accessibility" and include the page, browser, assistive technology, and task that did not work when you can do so safely.

Do not include passwords, session values, installation identifiers, or other private information in a public issue. If GitHub itself prevents you from using the reporting route, contact the maintainer through the n3try GitHub profile so an alternate route can be arranged.
